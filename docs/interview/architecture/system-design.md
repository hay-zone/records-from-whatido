# 系统设计

## 1. 系统设计原则

### 可扩展性
系统能够处理增长的负载。

### 高可用性
系统持续可用,最小化停机时间。

### 可维护性
易于理解、修改和扩展。

### 性能
快速响应用户请求。

## 2. 常见架构模式

### 分层架构
- 表现层
- 业务逻辑层
- 数据访问层

### MVC/MVVM
前端架构模式。

### 事件驱动架构
通过事件进行组件间通信。

## 3. 高可用方案

### 负载均衡
- Nginx
- HAProxy
- LVS

### 缓存
- Redis
- Memcached
- CDN

### 数据库
- 主从复制
- 读写分离
- 分库分表

### 容灾
- 多机房部署
- 异地灾备
- 限流降级

- 设计短链接服务
