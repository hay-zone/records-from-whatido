# 微服务架构

## 1. 什么是微服务

将单体应用拆分为一组小型服务,每个服务运行在独立的进程中。

### 特点
- 服务独立部署
- 服务独立扩展
- 松耦合
- 去中心化治理

## 2. 核心组件

### 服务注册与发现
- Consul
- Eureka
- Nacos

### API 网关
- Kong
- Zuul
- Spring Cloud Gateway

### 配置中心
- Apollo
- Spring Cloud Config
- Nacos

### 服务通信
- REST API
- gRPC
- 消息队列

## 3. 优缺点

### 优点
- 独立部署
- 技术栈灵活
- 易于扩展
- 容错隔离

### 缺点
- 分布式复杂性
- 数据一致性
- 运维成本高
- 测试复杂

## 4. 最佳实践

- 服务拆分要合理
- 使用容器化部署
- 实施 CI/CD
- 完善的监控和日志
- 服务熔断和降级
